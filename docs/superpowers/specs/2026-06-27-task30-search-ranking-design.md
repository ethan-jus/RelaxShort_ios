# Task30 Search 榜单交互设计

## 目标

Search 默认页使用真实后端榜单数据，并实现接近参考产品的横向露出式分页：当前榜单完整可读，右侧露出下一榜单的一部分，提示用户可以左右滑动。

本设计同时约束榜单标题、数据映射、卡片背景和交互状态，避免 UI 与接口语义脱节。

## 产品语义

三个榜单标题是固定产品文案，通过 iOS 本地化资源维护，不由后端动态配置：

| 固定标题 | 后端 `type` | 含义 |
| --- | --- | --- |
| Top Searched | `trending` | 搜索热度榜 |
| Most Trending | `popular` | 综合热播榜 |
| New Releases | `new` | 新上架榜 |

后端负责返回榜单内容和顺序；iOS 不得将同一份数据按评分、播放量或 ID 本地排序成多个榜单。

## 横向分页

不使用全宽 `TabView`。使用 iOS 17 的横向 `ScrollView`、`scrollTargetLayout()`、`scrollTargetBehavior(.viewAligned)` 和 `scrollPosition(id:)` 实现分页。

每个榜单页面宽度小于视口：

- 当前榜单宽度固定使用容器宽度的约 84%，不使用固定点数或 `min/max` 限制。
- 左侧内容边距约为容器宽度的 4%，页间距约为容器宽度的 2.5%。
- 右侧自然露出下一榜单约 9%–11%，至少可见下一页的排名角标和部分封面。
- 露出区域应至少能看到下一页的排名角标和部分封面。
- 第一页左侧与页面边距对齐。
- 最后一页允许完整吸附，尾部保留与页面边距一致的空间。

点击标题与手势滑动共用唯一的 `selectedTheme`：

- 点击标题时，分页容器动画滚动到目标榜单。
- 手势吸附到新榜单后，标题选中态同步更新。
- 不维护第二套选中索引。
- 不增加三个榜单纵向滚动位置的持久化状态；切换返回时允许回到顶部。

## 榜单卡片

统一使用 `SearchRankCardView`，卡片高度、封面尺寸、文字层级和热度区域保持一致。

卡片视觉规则：

- 每个榜单只有第 1–3 名使用主题渐变背景。
- 第 4 名及以后使用统一近黑背景，不使用主题渐变。
- Top Searched 前三名：低饱和暗酒红渐变至近黑。
- Most Trending 前三名：低饱和暗棕渐变至近黑。
- New Releases 前三名：低饱和暗青绿渐变至近黑。
- 渐变左侧使用低饱和主题色，右侧必须衰减至纯黑。
- 前三名排名角标分别使用 Logo 红、低饱和橙、低饱和金。
- 排名角标与封面的左边和上边完全贴齐，不保留内边距。
- 排名角标背景从左侧实色渐变到右侧较浅颜色。
- 排名角标只有左上角使用 `DB.posterRadius`，其余三个角保持直角。
- 第 4 名以后只使用白色排名文字和轻微暗底。
- 当前页必须完整显示封面、两行标题、单行分类和 tags、热度。
- 热度采用尾部固定区域，长标题不能挤掉热度。
- 热度火焰和数值均使用白色；火焰约 14pt，数值约 14pt bold。
- 卡片圆角和封面圆角使用现有 `DB.posterRadius`。

## 页面结构

`SearchView`：

- 管理返回按钮、搜索输入、搜索历史、搜索结果状态和播放器导航。
- 顶部搜索栏避开安全区，不使用固定屏幕宽度。
- 搜索框高度与 Home 搜索框共用 36pt 规范，禁止 Search 单独使用 44pt。
- 键盘提交写入历史。
- 输入为空且存在本地历史时，显示 `Recent Searches` 和历史词。
- 输入为空且本地历史为空时，读取 `/api/v2/search/default` 的 `suggestions`，显示 `Trending Searches`。
- 热门搜索词不得伪装成本地历史；只有用户执行搜索后才写入历史。
- 关键词搜索继续使用真实 `/api/v2/search` 和 cursor 分页。

`SearchDefaultView`：

- 接收历史记录和回调，不直接读写 `UserDefaults`。
- 展示固定标题和横向露出式榜单分页。
- 只负责默认页布局，不负责网络 DTO 解析。

`SearchDefaultViewModel`：

- 并发请求 `trending`、`popular`、`new`。
- 通过 `SearchRepositoryProtocol` 读取 `/api/v2/search/default` 的真实 suggestions。
- suggestions 失败不阻断三个榜单展示，记录 DEBUG 日志并将热门词降级为空。
- 保留接口返回顺序并生成展示排名。
- 不根据 `useRealAPI` 编写额外业务分支；Mock/Real 由注入的 Repository 决定。

`SearchRankTheme`：

- 集中维护固定标题的本地化 key、后端 type、前三名渐变颜色和普通卡片颜色。
- 三个标题使用 16pt，选中标题 semibold，未选中标题 regular，并保持单行。

## 搜索状态

搜索必须区分：

1. 尚未搜索。
2. 首次加载中。
3. 已有结果。
4. 搜索完成但为空。
5. 首屏请求失败。
6. 加载更多失败。

请求需记录规范化后的当前 query。旧请求晚返回时不能覆盖新查询结果，也不能错误结束新请求的 loading。

错误处理：

- 首屏失败显示本地化错误文案和 Retry。
- 有旧结果时请求失败保留旧结果。
- 加载更多失败保留已有结果，并允许后续再次触发。
- DEBUG 环境记录错误，不输出隐私数据。

## 多语言

新增 Search 文案必须进入现有九个资源目录：

- Base
- en
- zh-Hans
- zh-Hant
- es
- pt
- ja
- ko
- ar

禁止在 `RelaxShort/Localizable.strings` 创建脱离 `.lproj` 的资源文件。榜单标题固定不等于英文硬编码，所有展示文案必须通过 `L10n`。

`Trending Searches`、清除搜索输入和榜单无障碍格式也必须具有九种语言资源。

## 响应式要求

- 小屏、标准屏、Pro Max 使用同一套比例和上下限约束。
- 三个标题保持单行，均匀分配点击区域。
- 横向页面宽度和间距按容器比例计算，不得针对单一机型写死。
- 横向露出区域不得挤压当前页的热度区域。
- 长标题最多两行，卡片高度固定。
- 阿拉伯语至少保证文字不溢出、按钮可点击；系统返回方向遵循布局方向。

## 验收

代码验收：

- `SearchRankTheme.swift` 和 `SearchRankCardView.swift` 是独立文件并加入 Xcode Sources。
- 不存在同名内联副本。
- 九种语言资源 key 完整。
- `git diff --check` 无错误。
- iPhone 17 Pro Max 构建成功。

功能验收：

- 三个榜单分别请求正确的真实 API type。
- 点击标题和左右滑动同步。
- 每次滑动吸附到一个榜单，不停在任意中间位置。
- 当前榜单完整，右侧稳定露出下一榜单的部分封面。
- 每个榜单只有前三名使用主题渐变。
- 前三名背景右侧为纯黑，排名角标仅左上角有圆角并贴齐封面边缘。
- 热度使用更醒目的白色大号图标和粗体数值。
- 有历史时显示 Recent Searches；无历史时显示后端 Trending Searches。
- Search 与 Home 搜索框高度一致。
- 搜索、历史、空态、错误 Retry、分页和卡片跳转正常。
- 小屏、标准屏和大屏无文字溢出、遮挡或安全区问题。
